## Session Document Structure

Each session document follows this structure:

1. **Phase 0: Preparation** - Environment setup, context gathering
2. **Phase N: Implementation** - Ordered implementation phases with checkboxes
3. **Session Notes** - Progress log, decisions, blockers
4. **Quick Reference** - Commands, key files, conventions

## Using Sessions

### For Agents

1. Read the session document completely before starting
2. Check off items as they are completed
3. Add dated entries to the Progress Log
4. Document decisions and blockers
5. Update related documentation as you go

### Starting a New Session

1. Create a new markdown file in this directory
2. Use an existing session as a template
3. Break work into logical phases
4. Include verification steps after each phase
5. Add to this INDEX.md
